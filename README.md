# 言库·轻舟

言库·轻舟（包名 `言舟`，仓库 `yanxu-sqlite`）是在 `言库` 统一协议之上的 SQLite 实现。默认后端是 SQLite 官方 `sqlite3` CLI：言序负责安全绑定、结果归一、事务脚本、保存点和迁移，宿主进程负责真正打开 SQLite 数据库。

主要能力：

- SQLite 文件连接、参数化 SQL、可复用预编译语句外观；
- 统一查询结果、影响行数、末插入号和结构化安全执行结果；
- 单进程原子事务批、立即事务、保存点、回滚至与释放；
- 批量语句、迁移登记、待升级计划、升级与按次数回退；
- 可注入执行器，便于测试或替换为原生扩展驱动；
- SQL 感知占位符扫描，忽略引号内的 `?` 并严格检查参数数量。

## 使用言包安装

```sh
yanbao add sqlite --package 言舟 --version "^0.1"
```

等价清单：

```toml
[依赖]
言舟 = { git = "https://github.com/yanxulang/yanxu-sqlite.git", 修订 = "main", 版 = "^0.1" }
```

言包会自动安装 `yanxu-db（言库）`。

## 快速开始

请先安装 SQLite 3.38+ 并确保 `sqlite3` 在 `PATH` 中：

```yanxu
引「包:言舟」为 言舟；

定 数据库 为 言舟.打开（「应用.db」）；
数据库.执行（「CREATE TABLE IF NOT EXISTS users(id INTEGER PRIMARY KEY, name TEXT)」，【】）；
数据库.预编译（「INSERT INTO users(name) VALUES (?)」）.执行（【「子衿」】）；
言 数据库.查询（「SELECT * FROM users WHERE name = ?」，【「子衿」】）.全部（）；
数据库.关闭（）；
```

自定义程序路径和超时可用 `打开为(数据库路径, 程序, 超时毫秒)`。

## 事务边界

标准 `进程.执行` 没有持久标准输入，因此 CLI 后端把事务内写操作安全绑定后缓存，在 `提交()` 时一次交给同一个 SQLite 进程执行；这保证原子性与保存点语义。事务批不在提交前返回查询结果，应提交后通过连接查询。注入持久原生执行器时可在更高层实现交互式事务。

使用默认后端的顶层应用必须显式授予 `进程 = true`；依赖不会替应用提升权限。完整接口见 [docs/API.md](docs/API.md)。
