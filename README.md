# 言库·轻舟

言库·轻舟（包名 `言舟`，仓库 `yanxu-sqlite`）是建立在 `言库` 1.x 协议上的生产级 SQLite 驱动。默认后端是随包发布的 ABI v2 原生扩展，直接在进程内使用固定版本 SQLite；系统 `sqlite3` CLI 仅作为显式兼容后端保留。

当前稳定版本为 1.0.0。

## 主要能力

- 文件、内存、临时、URI 和只读数据库；
- WAL、外键、同步模式与 busy timeout 配置；
- 原生参数绑定、可复用预编译语句和资源级关闭；
- 交互式事务、立即事务、保存点、回滚至和释放；
- 影响行数、最后插入号、列元数据和结构化驱动错误；
- 表、列、索引、索引列与外键反射；
- 带校验和、状态、干运行、原子执行和写锁复查的迁移；
- 规范言据文本与 SQLite JSON1 原生结构策略；
- 参数化 JSON 路径存在、等值、数字比较和文字包含条件；
- macOS、Linux、Windows 的 x86-64 与 ARM64 原生制品；
- 独立 `sqlite3` CLI 兼容入口和可注入测试执行器。

## 安装

使用言包安装稳定 1.x：

```sh
yanbao add sqlite --package 言舟 --version "^1.0"
```

等价清单：

```toml
[依赖]
言舟 = { git = "https://github.com/yanxulang/yanxu-sqlite.git", 修订 = "v1.0.0", 版 = "^1.0" }

[权限]
文件 = ["."]
原生扩展 = true
```

应用必须自行授予实际数据库路径对应的文件权限和原生扩展权限；依赖不会传递授权。仅使用内存数据库时可将文件权限保持为空。使用 CLI 兼容后端还须设置 `进程 = true`。

## 五分钟示例

```yanxu
引「包:言舟」为 言舟；

定 数据库 为 言舟.打开（「应用.db」）；

数据库.执行（
    「CREATE TABLE IF NOT EXISTS users(id INTEGER PRIMARY KEY, name TEXT NOT NULL)」，
    【】
）；

定 插入 为 数据库.预编译（「INSERT INTO users(name) VALUES (?)」）；
插入.执行（【「子衿」】）；
插入.关闭（）；

定 用户 为 数据库.查询（
    「SELECT id, name FROM users WHERE name = ?」，
    【「子衿」】
）.首行（）；

言 用户；
数据库.关闭（）；
```

SQL 模板与参数必须分开传入。表名、列名等 SQL 标识符不能作为值参数绑定，应由受信任代码通过方言引用 API 构造。

## 打开方式

```yanxu
定 内存 为 言舟.打开内存（）；
定 临时 为 言舟.打开临时（）；
定 只读 为 言舟.打开只读（「已有.db」）；
定 共享 为 言舟.打开URI（「file:shared?mode=memory&cache=shared」）；

定 已配置 为 言舟.打开配置（{
    「路径」：「应用.db」，
    「日志模式」：「WAL」，
    「同步模式」：「NORMAL」，
    「外键」：真，
    「忙碌超时毫秒」：5000
}）；
```

文件 URI 为了与言序文件权限保持同一边界，不接受 authority、百分号编码、片段、重复 `mode` 或多个查询串。支持的 `mode` 为 `ro`、`rw`、`rwc` 和 `memory`。

## 原生与 CLI

`打开`、`打开配置`、`打开内存`、`打开临时`、`打开URI` 和只读入口使用原生后端。`打开CLI` 与 `打开为` 显式使用系统 `sqlite3`。

CLI 后端要求 SQLite 3.38+，提供兼容查询、执行、批量和提交时事务脚本，但不提供真正的原生语句句柄；事务提交前也不能返回查询结果。默认原生制品内置 SQLite 3.53.2，不依赖系统 SQLite 安装。

## 错误处理

驱动与言库错误均使用稳定代码开头。常见类别包括：

- `YANKU_SQLITE_*`：SQLite 执行、约束、忙碌、只读和资源错误；
- `YANZHOU_*`：打开配置、URI、绑定、事务、迁移和言据边界错误；
- `YANKU_CAPABILITY`：当前后端没有声明所需能力；
- `未获文件权限`、`未获进程权限`、`未获原生扩展权限`：顶层应用授权不足。

程序应按稳定代码分类，不应依赖完整中文消息。

## 兼容与限制

- 最低言序版本：1.1.12；
- 稳定依赖：言库 1.x；
- 原生制品：macOS、Linux、Windows，x86-64 与 ARM64；
- JSON1 按运行时探测，CLI 后端不声明 JSON1；
- 言舟依赖 `标准:原生`，须使用字节码 VM/YXB，树解释器不能载入；
- 原生查询单次最多返回 100,000 行、1,024 列和 16 MiB 结果；
- 原生 SQL 最大 1 MiB，参数最多 65,536 个；
- CLI 事务查询须在提交后由连接重新执行；
- 当前 API 不提供异步取消或连接池，连接池由 `言库` 上层组合。
