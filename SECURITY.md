# 安全策略

## 支持版本

| 版本 | 安全更新 |
| --- | --- |
| 1.x | 是 |
| 0.1.x | 否 |

## 报告漏洞

请使用 GitHub 仓库的私有安全报告功能。报告应包含受影响版本、平台、最小复现、实际影响和建议缓解方式。请勿在公开议题中提交生产数据库、个人数据、访问令牌、未修复利用代码或可识别文件路径。

## 发布响应

维护者确认问题后会评估受影响版本和制品，准备修复、测试六平台构建，并以补丁版本发布。若原生制品受影响，Release 会列出替换后的 SHA-256；旧制品不会静默覆盖同一标签。

## 安全边界摘要

- 默认原生后端通过 SQLite 参数 API 绑定值；
- CLI 兼容后端仅在普通 SQL 上下文替换匿名 `?`；
- 文件和 URI 打开前触发顶层应用路径权限检查；
- 原生扩展与宿主进程同权限，不是沙箱；
- 原生制品按目标、大小和 SHA-256 锁定；
- URI 拒绝 authority、百分号编码、片段、重复 mode 和歧义查询串；
- 查询、SQL、参数、行列和结果字节均有限额；
- 迁移以校验和检测漂移，并在写锁内重新规划。

完整边界见 [docs/SECURITY_MODEL.md](docs/SECURITY_MODEL.md)。
