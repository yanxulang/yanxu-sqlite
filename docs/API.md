# 言库·轻舟 API

本文记录 `言舟` 1.x 的公开面。未列出的私有法、ABI 内部结构和制品文件名不是应用 API。

## 常量

- `版本：文`：包 API 版本；
- `SQLite最低版本：文`：CLI 兼容后端最低 SQLite 版本，当前为 `3.38.0`；
- `最大安全整数：数`：跨言序和 SQLite 保持精确的整数上限；
- `最大迁移SQL字符：数`：单段升级或降级 SQL 字符上限；
- `最大迁移数：数`：单个迁移集合数量上限。

## 打开入口

### `打开(数据库路径)`

以原生后端打开可创建的文件数据库。默认外键开启、日志模式为 WAL、busy timeout 为 5,000 毫秒。

### `打开配置(配置)`

原生后端完整入口。配置键：

| 键 | 类型 | 默认 | 说明 |
| --- | --- | --- | --- |
| `路径` | 文 | 必填 | 文件路径、`:memory:` 或 `file:` URI；临时模式不填 |
| `临时` | 理 | 假 | 使用 SQLite 空文件名临时数据库 |
| `URI` | 理 | 路径以 `file:` 开头时为真 | 启用 SQLite URI 解析 |
| `只读` | 理 | `mode=ro` 时为真 | 使用只读打开旗标 |
| `创建` | 理 | 由只读和 URI mode 推导 | 是否允许创建文件 |
| `外键` | 理 | 真 | 设置 `PRAGMA foreign_keys` |
| `忙碌超时毫秒` | 数 | 5000 | 1 至 604,800,000 |
| `日志模式` | 文 | 文件为 `WAL`，内存为 `MEMORY` | `DELETE/TRUNCATE/PERSIST/MEMORY/WAL/OFF` |
| `同步模式` | 文 | `NORMAL` | `OFF/NORMAL/FULL/EXTRA` |

冲突配置会在打开前拒绝，例如只读同时创建、临时同时给路径、URI 不是 `file:`、URI mode 与显式旗标不一致。

### 便捷入口

- `打开内存()`：独立 `:memory:` 数据库；
- `打开临时()`：SQLite 管理生命周期的临时数据库；
- `打开URI(URI)`：启用 URI；`mode=ro` 自动推导只读；
- `打开只读(数据库路径)`：只读打开已有文件；
- `打开只读URI(URI)`：只读 URI；
- `打开CLI(数据库路径)`：系统 `sqlite3`、30 秒调用超时；
- `打开为(数据库路径, 程序, 超时毫秒)`：自定义 CLI 程序和超时；
- `以执行器(执行器)`：注入言库请求/响应执行器；
- `执行器工厂(配置)`：原生驱动工厂；
- `CLI执行器工厂(配置)`：CLI 驱动工厂，键为 `路径`、`程序`、`超时毫秒`。

## `SQLite连接`

### 查询与执行

- `执行(SQL, 参数)`：返回言库执行结果，含影响行数与最后插入号；
- `查询(SQL, 参数)`：返回言库查询结果和列元数据；
- `安全执行(SQL, 参数)`：返回 `{成功, 结果, 错误}`，错误为 `SQLite错误.转典()`；
- `批量执行(各语句)`：在立即事务中执行 SQL 文字或 `{SQL, 参数}` 典列，成功返回项数；
- `预编译(SQL)`：原生后端返回 `SQLite原生预编译`，兼容后端返回 `SQLite预编译`。

### 事务

- `开始事务()`：原生后端执行 `BEGIN`；CLI 后端创建延迟事务批；
- `立即事务()`：原生后端执行 `BEGIN IMMEDIATE`；CLI 后端创建立即事务批。

### 反射

- `表清单()`：返回 main/temp 中非 SQLite 内部表与视图；
- `表结构(名称)`：返回存在、模式、种类、列数、无行号、严格、创建 SQL、列、索引和外键；
- `列信息(名称)`、`索引信息(名称)`、`外键信息(名称)`：对应结构子集。

列信息含序号、名称、声明类型、不可空、默认值、主键序位和隐藏标记。索引含唯一、来源、部分索引标记及嵌套索引列。外键含目标表/列及更新、删除、匹配动作。

### 言据与能力

- `能力()`：返回言库驱动能力；
- `言据写入值(值, 策略)`：策略为 `规范文本` 或 `原生结构`；
- `言据读取值(值, 策略)`：恢复等价言据值；
- `言据路径(列名, 各段)`：JSON1 可用时返回参数化路径构建器；
- `信息()`：原生后端返回 SQLite 版本、JSON1、日志模式、外键、只读、URI、临时、busy timeout 与资源限制。

### 生命周期

- `是否关闭()`：连接关闭状态；
- `关闭()`：幂等关闭，原生父资源同时关闭子语句资源；
- `方言()`：SQLite 方言；
- `原始连接()`：底层言库连接。

`设信息器`、`设事务工厂`、`设语句工厂`、`设反射器` 是驱动组合扩展点。普通应用通常不直接调用。

## 预编译语句

### `SQLite原生预编译`

- 只读域 `SQL`；
- `执行(参数)`、`查询(参数)`；
- `执行带选项(参数, 选项)`、`查询带选项(参数, 选项)`；
- `信息()`：SQL 与参数数；
- `参数数()`；
- `已绑定(参数)`：仅用于诊断展示，不用于实际原生执行；
- `关闭()`、`是否关闭()`。

参数数量由 SQLite 编译后的语句报告。连接关闭会级联失效语句；语句关闭后调用返回 `YANZHOU_STATEMENT_CLOSED` 或原生资源错误。

### `SQLite预编译`

CLI/自定义执行器的兼容外观：`执行`、`查询`、`已绑定`、`关闭`、`是否关闭`。它保存 SQL 模板，但每次调用仍由兼容执行器绑定和执行。

## 事务对象

### `原生事务`

- `执行(SQL, 参数)`、`查询(SQL, 参数)`；
- `保存点(名称)`、`回滚至(名称)`、`释放点(名称)`；
- `提交()`、`回滚()`、`是否活跃()`。

保存点名称由原生层按 SQLite 标识符规则引用。事务结束后继续调用返回状态错误。

### `事务批`

CLI/自定义执行器兼容对象：`执行` 缓存已安全绑定的 SQL；`保存点`、`回滚至`、`释放点` 维护后进先出状态；`脚本()` 返回完整事务脚本；`提交()` 一次执行；`回滚()` 丢弃缓存。

`查询` 在提交前固定返回 `YANZHOU_TRANSACTION_QUERY`。应提交后通过连接查询。

## JSON1 路径

`JSON路径(各段)` 将文字键和非负整数索引构造成 SQLite JSON 路径。最多 64 段、单文字段 512 字符、总长度 4,096 字符。

`SQLite言据路径(列名, 各段)` 的方法：

- `取值()`；
- `存在()`、`不存在()`；
- `等值(值)`、`非等值(值)`；
- `大于值(数)`、`至少(数)`、`小于值(数)`、`至多(数)`；
- `文字包含(文)`。

每个方法返回 `SQLite表达式`。表达式公开 `SQL`，并通过 `参数()` 返回复制后的参数列；`转典()` 返回 `{SQL, 参数}`。路径和比较值始终作为参数，不拼接到 SQL。

## 迁移

### `迁移`

`迁移(版本号, 名称, 升级SQL, 降级SQL)` 创建描述。版本必须为正安全整数，名称 1 至 256 字符，升级 SQL 非空。公开只读域为版本号、名称、升级/降级 SQL 和 SHA-256 校验和；方法为 `是否可逆()` 与 `转典()`。

`迁移计划(各迁移, 已应用版本)` 验证严格递增顺序并返回待升级迁移。

### `迁移器`

`默认迁移器(连接)` 使用 `_yanxu_migrations`；也可构造 `迁移器(连接, 表名)`。

- `准备()`：创建登记表并兼容增加 checksum 列；
- `已应用()`：按版本升序读取；
- `待升级(各迁移)`；
- `状态(各迁移)`：每项含迁移描述和 `已应用/待升级` 状态；
- `检查(各迁移)`：返回有效性及统计；
- `干运行升级(各迁移)`；
- `升级(各迁移)`；
- `干运行回退(各迁移, 次数)`；
- `回退(各迁移, 次数)`；
- `采用旧校验和(各迁移)`：显式为旧登记补校验和。

原生升级与回退使用 `BEGIN IMMEDIATE`，取得写锁后重新读取并规划，所有 SQL 与登记在同一事务内完成。内容漂移、名称漂移、缺失定义和不可逆回退均使用稳定错误拒绝。

## 桥与错误

`原生桥`、`CLI桥` 是公开组合边界，主要供驱动适配和测试使用。直接使用时调用方必须维护连接与语句资源生命周期。所有公开类均以 `初始化` 法接收各自构造参数。

`CLI桥.执行请求(请求)` 接受言库函数执行器信封，并通过配置的程序、数据库路径和超时执行。`原生桥` 提供：

- `执行请求(请求)`、`信息()`、`表清单()`、`表结构(名称)`；
- `准备语句(连接, SQL)`；
- `语句请求(语句句柄, 请求)`、`语句信息(语句句柄)`、`关闭语句(语句句柄)`；
- `开始事务(连接, 立即)`、`提交事务()`、`回滚事务()`；
- `保存点(名称)`、`回滚至(名称)`、`释放点(名称)`；
- `是否关闭()`。

`SQLite错误` 公开代码、消息、SQL、参数、程序和踪迹，并提供 `转典()`。`安全执行` 会构造该兼容错误；普通言库执行错误仍按言库稳定错误返回。

## 兼容绑定工具

`SQL字面(值)` 与 `绑定(SQL, 参数)` 仅服务 CLI 兼容后端和审计展示。扫描器忽略单/双引号、反引号、方括号标识符、行注释和块注释中的 `?`，并严格匹配参数数。

生产默认原生后端不把参数转成 SQL 字面量，而是调用 SQLite 原生绑定 API。
