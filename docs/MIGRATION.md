# 从 0.1 迁移到 1.0

## 默认后端变化

0.1 的 `打开` 使用系统 `sqlite3` CLI；1.0 的 `打开` 使用 ABI v2 原生后端。需要保留旧执行模型时改用：

```yanxu
定 数据库 为 言舟.打开CLI（「应用.db」）；
```

原生后端不要求系统安装 SQLite，支持真正预编译资源和交互式事务。CLI 仍要求 SQLite 3.38+。

## 权限

顶层应用必须显式授予：

```toml
[权限]
文件 = ["数据库所在目录"]
原生扩展 = true
```

CLI 兼容后端另需 `进程 = true`。内存和安全内存 URI 不触发文件权限；文件名中出现 `mode=memory` 不会绕过权限。

## 事务行为

1.0 原生事务中的 `查询` 会立即返回结果。CLI 事务仍在提交前拒绝查询。依赖旧事务批 `脚本()` 的代码应显式使用 CLI/自定义执行器，不要假设原生事务公开脚本文本。

## 预编译语句

原生 `预编译` 返回持有资源的 `SQLite原生预编译`。使用完成后调用 `关闭()`；连接关闭会级联失效未关闭语句。不要长期缓存超过连接生命周期的语句对象。

## 迁移登记

1.0 为 `_yanxu_migrations` 增加 `checksum`。首次读取旧表时，缺失校验和会返回 `YANZHOU_MIGRATION_LEGACY`。审计旧迁移名称与内容后执行：

```yanxu
迁移器.采用旧校验和（各迁移）；
```

以后修改已应用迁移会返回校验和漂移错误。应新增更高版本迁移，而不是编辑历史。

## 依赖

清单改为言舟 `^1.0` 并锁定言库 1.x。更新锁后同时运行源码消费者与 Release YXB，确认目标原生制品已被选择。
