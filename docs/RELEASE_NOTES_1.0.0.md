# 言舟 1.0.0

言舟 1.0.0 将早期的 `sqlite3` 进程适配器升级为可用于生产的 SQLite 驱动，并保持显式 CLI 兼容入口。默认后端通过言序 ABI v2 在进程内使用固定版本 SQLite。

## 主要能力

- 文件、内存、临时、URI 和只读数据库，以及 WAL、外键、同步模式和 busy timeout；
- SQLite 原生参数绑定、可复用预编译资源、批量执行、影响行数和最后插入号；
- 交互式事务、立即事务、保存点、安全关闭和父连接资源级联；
- 表、列、索引、索引列和外键反射；
- 带校验和、状态、检查、干运行、原子登记和写锁复查的迁移；
- 规范言据文本、JSON1 能力探测、原生结构存储和参数化路径条件；
- macOS、Linux、Windows 的 x86-64 与 ARM64 固定原生制品。

## 安全

- 默认后端使用 SQLite 参数 API，不把用户值拼接进 SQL；
- 文件 URI 拒绝 authority、百分号编码、片段、多个查询串和重复 `mode`；
- 原生制品按平台、架构、普通文件、大小和 SHA-256 校验；
- SQL、参数、结果行列和结果字节均有硬限制；
- 驱动错误使用稳定代码分类，不回显数据库密码或未受控原生错误文本。

## 破坏性变化与迁移

`打开` 现在选择原生后端。仍需系统 `sqlite3` 的应用应改用 `打开CLI`；需要指定程序路径时使用 `打开为`。顶层应用必须授予实际数据库路径的文件权限和原生扩展权限。完整迁移步骤见 [MIGRATION.md](MIGRATION.md)。

## 兼容性与依赖

最低言序版本为 1.1.12，稳定依赖为言库 1.x，清单固定言库 `v1.0.0`。默认原生后端内置 SQLite 3.53.2；CLI 后端要求系统 SQLite 3.38 或更新版本。原生后端不需要网络、环境或进程权限；CLI 后端另需进程权限。

## 验证

发布门禁包含 7 个言序源码文件、2 个规格、1 个示例、3 个独立消费者、7 个 Rust 单元测试、在线/离线锁验证、4 个 Release YXB 构建、3 个 YXB 执行，以及六种目标制品的格式、架构、ABI 符号、大小和 SHA-256 检查。原生消费者在 Ubuntu、macOS、Windows runner 上执行。

## 已知限制

- 言舟依赖 `标准:原生`，不能由树解释器载入；
- CLI 兼容事务在提交前不能返回事务内查询结果，也不提供真正的预编译句柄；
- JSON1 按运行时探测，CLI 后端不声明 JSON1；
- 当前驱动不提供异步取消或连接池，连接池由言库上层组合；
- 原生查询单次最多返回 100,000 行、1,024 列和 16 MiB 结果。
