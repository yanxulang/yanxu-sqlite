# 安全模型

## 受保护资产

- 数据库文件及 WAL/SHM 伴随文件；
- SQL 参数中的业务和个人数据；
- 原生连接、事务和预编译资源；
- 迁移历史与结构完整性；
- 发布制品供应链。

## 信任边界

应用负责数据库路径授权、SQL 模板和动态标识符白名单；言舟负责打开配置验证、参数绑定、资源预算、事务状态、迁移一致性和驱动能力；SQLite 负责文件格式、锁和 SQL 执行。

原生扩展与宿主进程拥有相同操作系统权限。言序的原生制品摘要门禁防止目标或字节被静默替换，但不隔离恶意原生代码。

## 文件与 URI

ABI v2 的原生权限回调没有路径级文件能力，因此公开打开入口先调用 `标准:文件.存在` 触发顶层路径授权，再进入原生层。临时、`:memory:`、`file::memory:` 和唯一 `mode=memory` URI 不访问应用文件，可不要求文件权限。

为了避免权限检查路径与 SQLite 实际路径解码不一致，文件 URI 拒绝 authority、百分号编码、片段、多个查询串和重复 mode。原生层重复验证 URI mode 与只读/创建旗标。

应用不应绕过公开 API 直接构造 ABI 调用。原生扩展本身不是文件沙箱。

## SQL 注入

原生后端只把 SQL 模板传给 SQLite 编译器，值通过参数 API 绑定。JSON 路径和值也作为参数；表和列名通过方言引用。

CLI 兼容绑定器识别 SQLite 引号、反引号、方括号、行注释和块注释，只替换普通上下文中的匿名 `?`。它不能使不可信 SQL 模板安全。调用方不得把用户输入拼成 SQL、迁移 SQL、排序表达式或标识符。

## 资源限制

原生层限制 SQL、参数、行、列和结果字节，并拒绝无效 UTF-8、超大值和不支持类型。连接关闭级联关闭语句资源；显式关闭和析构路径均测试泄漏统计。

busy timeout 只控制 SQLite 锁等待，不是总查询截止时间。当前驱动不声称可以抢占正在运行的查询。

## 迁移

迁移校验和覆盖版本、名称、升级和降级 SQL。已应用定义缺失、名称变化、内容变化和旧空校验和均拒绝自动继续。升级与回退在立即事务中重读状态，失败时原子回滚。

迁移 SQL 是受信任发布代码。驱动不会尝试静态判断任意 SQL 是否破坏数据。

## 言据与 JSON

规范文本和原生结构都受言库交换大小限制。原生 JSON 不是言据文本；读取时必须解析并恢复言据值。JSON1 不可用时路径查询明确返回能力错误，不会加载全表模拟。

## 制品

六个平台制品固定 SHA-256、大小和目标。构建使用锁定 Cargo 依赖与 bundled SQLite。macOS 制品检查安装名和签名，Windows 静态 CRT，所有制品检查 ABI v2 导出。
